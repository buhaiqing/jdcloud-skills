# 金融科技架构方案（京东云版）

## 架构总览

```mermaid
graph TB
    subgraph 接入层
        USR[终端用户]
        WAF[Web 应用防护]
        CLB[应用型 CLB]
    end
    subgraph 应用层
        APP1[业务服务 A (多 AZ)]
        APP2[业务服务 B (多 AZ)]
    end
    subgraph 异步层
        JCQ[消息队列 JCQ (事务消息)]
    end
    subgraph 数据层
        MYSQL[JCS for MySQL 主备 + 跨区域灾备]
        REDIS[JCS for Redis (账务缓存)]
    end
    subgraph 审计
        AUDIT[ActionTrail 全量审计]
        KMS[KMS 密钥托管]
    end

    USR --> WAF --> CLB --> APP1
    USR --> WAF --> CLB --> APP2
    APP1 --> JCQ --> MYSQL
    APP1 --> REDIS
    APP2 --> JCQ
    MYSQL --> KMS
    CLB --> AUDIT
```

## 合规与安全设计

### 等级保护要求
- **等保 2.0 三级** 起步
- **金融行业** 额外满足 JR/T 0167-2018 等规范
- **数据本地化**: 用户数据不出境，关键数据不出域

### 核心安全措施
- **WAF + 高防 IP**: 抵御 Web 攻击和大流量 DDoS
- **ActionTrail 全量审计**: 保留 ≥ 180 天，可追溯到人/操作/API
- **KMS 密钥托管**: 数据库 TDE 加密、应用层字段加密
- **IAM 最小权限**: 财务/运维/审计账号分离
- **数据库审计**: 记录所有 SQL 操作，识别越权访问

### 资金安全
- **分布式事务**: JCQ 事务消息 + 业务幂等
- **对账系统**: T+1 自动化对账，差异人工复核
- **风控前置**: 实时反欺诈引擎，命中规则直接拒绝
- **熔断降级**: 关键路径多级熔断，故障 30s 内自动恢复

## 容灾与 RTO/RPO

| 等级 | 部署模式 | RTO | RPO | 适用 |
|------|---------|-----|-----|------|
| 单活 | 同城单 AZ | 不可用 | 0 | 仅测试 |
| 同城双活 | 同城 2 AZ | < 5 min | < 1 min | 一般业务 |
| 同城双活 + 异地灾备 | 2+1 | < 30 min | < 5 min | 核心交易 |
| 两地三中心 | 同城双活 + 异地独立 | < 60 min | < 1 min | 资金核心 |

## 数据层选型

| 业务 | 推荐 | 理由 |
|------|------|------|
| 账户/账务 | JCS for MySQL 主备 + 强同步 | 强一致、ACID |
| 流水/历史 | JCS for MySQL 分库分表 + 对象存储归档 | 写入量大、冷热分层 |
| 排行榜/限流 | JCS for Redis Cluster | 高性能、低延迟 |
| 日志/审计 | JCS for Elasticsearch | 全文检索、聚合分析 |

## 成本估算（月）

- **中小银行核心系统**: ¥300,000 - ¥800,000
- **互联网支付平台**: ¥150,000 - ¥500,000
- **消费金融业务系统**: ¥100,000 - ¥300,000
- **保险核心系统**: ¥500,000+

> 金融场景对 RTO/RPO 极敏感，**成本不是首要考虑**，合规和安全优先。
