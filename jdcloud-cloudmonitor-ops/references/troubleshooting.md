# 云监控故障排查指南

## 常见错误代码

| 错误代码 | 描述 | 解决方案 |
|----------|------|----------|
| `InvalidParameter` | 参数无效或缺失 | 检查请求参数是否符合 API 要求 |
| `InvalidParameterValue` | 参数值无效 | 检查参数值格式和取值范围 |
| `MissingParameter` | 缺少必填参数 | 补充缺失的必填参数 |
| `ResourceNotFound` | 资源不存在 | 检查资源 ID 是否正确 |
| `MetricNotFound` | 监控项不存在 | 确认监控项名称是否正确 |
| `AlarmNotFound` | 告警规则不存在 | 检查告警规则 ID 是否正确 |
| `ServiceNotFound` | 服务不存在 | 确认服务代码是否正确 |
| `QuotaExceeded` | 超出配额限制 | 检查是否达到资源配额上限 |
| `RateLimitExceeded` | 请求频率超限 | 降低请求频率或申请提高限流 |
| `InternalError` | 内部错误 | 稍后重试或联系技术支持 |
| `Unauthorized` | 未授权 | 检查访问密钥和权限配置 |
| `RegionNotFound` | 区域不存在 | 确认区域 ID 是否正确 |

## 监控数据查询问题

### 问题1：查询不到监控数据

**现象**：调用 `describe-metric-data` 返回空数据。

**可能原因及解决方案**：

1. **资源未产生数据**
   - 新创建的资源可能还没有监控数据
   - 等待 5-10 分钟后重试

2. **时间范围不正确**
   - 检查 `start-time` 和 `end-time` 格式是否正确（ISO8601）
   - 确保查询的时间范围在数据保留期内（默认 15 天）
   ```bash
   # 正确的时间格式
   --start-time "2024-01-01T00:00:00Z"
   --end-time "2024-01-01T23:59:59Z"
   ```

3. **资源 ID 错误**
   - 确认 `resource-id` 参数值正确
   - 使用 `jdc vm describe-instances` 确认资源存在

4. **监控项名称错误**
   - 使用 `describe-metrics` 查询可用的监控项列表
   - 注意监控项名称区分大小写

### 问题2：监控数据延迟

**现象**：查询到的数据比当前时间延迟几分钟。

**解决方案**：
- 云监控数据采集和处理需要时间，通常有 3-5 分钟延迟
- 对于实时性要求高的场景，可使用 `last-downsample` 接口查询最新数据
- 如需更低延迟，考虑使用应用性能监控(SGM)或自定义监控

### 问题3：监控数据不连续

**现象**：监控数据曲线有断点或缺失。

**可能原因及解决方案**：

1. **资源状态变化**
   - 资源关机、重启期间不会产生监控数据
   - 检查资源运行状态

2. **网络问题**
   - 资源网络不通可能导致数据采集失败
   - 检查资源网络连接

3. **采集异常**
   - 联系京东云技术支持检查采集服务状态

## 告警规则问题

### 问题1：告警规则未触发

**现象**：监控指标已超过阈值，但告警未触发。

**排查步骤**：

1. **检查告警规则状态**
   ```bash
   jdc monitor describe-alarm --region-id cn-north-1 --alarm-id alarm-xxx
   ```
   - 确认规则状态为 `enabled`
   - 检查 `status` 字段是否为 `ALARM`

2. **检查阈值和比较运算符**
   - 确认阈值设置正确
   - 检查比较运算符方向（gt/lt/ge/le）
   - 注意数值类型（整数 vs 浮点数）

3. **检查连续周期数**
   - `evaluation-periods` 表示连续多少个周期满足条件才触发
   - 如需立即触发，设置为 1

4. **检查监控数据**
   ```bash
   jdc monitor describe-metric-data \
     --region-id cn-north-1 \
     --metric vm.cpu.util \
     --service-code vm \
     --resource-id i-xxx \
     --start-time "2024-01-01T00:00:00Z" \
     --end-time "2024-01-01T23:59:59Z"
   ```

### 问题2：告警通知未收到

**现象**：告警已触发，但未收到通知。

**排查步骤**：

1. **检查通知类型配置**
   - 确认 `notice-type` 包含需要的通知方式
   - 支持的类型：`sms`、`email`、`callback`

2. **检查联系人配置**
   - 确认 `contact-group-id` 正确
   - 检查联系人信息是否完整（手机号、邮箱）
   ```bash
   jdc monitor describe-alarm-contacts --region-id cn-north-1
   ```

3. **检查通知时段**
   - 确认告警触发时间在通知时段内
   - 检查 `notice-time` 配置

4. **检查通知周期**
   - `notice-period` 控制重复通知间隔
   - 如设置为 0，告警持续期间不会重复通知

5. **检查短信/邮件配额**
   - 确认账户短信/邮件配额未用完

### 问题3：告警频繁触发

**现象**：告警频繁在 ALARM 和 OK 状态间切换（告警震荡）。

**解决方案**：

1. **调整阈值**
   - 适当提高或降低阈值，避免临界值波动
   - 建议设置合理的缓冲区

2. **增加连续周期数**
   - 将 `evaluation-periods` 从 1 增加到 2 或 3
   - 避免瞬时波动触发告警

3. **使用数据聚合**
   - 增大 `period` 值，使用更长时间窗口的数据
   - 平滑瞬时波动

4. **设置告警抑制**
   - 配置 `notice-period` 控制重复通知频率
   - 避免告警风暴

## CLI 使用问题

### 问题1：认证失败

**现象**：返回 `Unauthorized` 或 `InvalidAccessKey` 错误。

**解决方案**：

1. **检查环境变量**
   ```bash
   echo $JDC_ACCESS_KEY
   echo $JDC_SECRET_KEY
   ```

2. **重新配置凭证**
   ```bash
   jdc config init
   ```

3. **检查密钥权限**
   - 确认 Access Key 有调用云监控 API 的权限
   - 检查密钥是否被禁用或删除

4. **检查系统时间**
   - 确保系统时间与标准时间同步
   - 时间偏差会导致签名验证失败

### 问题2：命令找不到

**现象**：执行 `jdc monitor` 提示命令不存在。

**解决方案**：

1. **检查 CLI 版本**
   ```bash
   jdc --version
   ```
   - 云监控功能需要 CLI 1.x 以上版本

2. **更新 CLI**
   ```bash
   pip install --upgrade jdcloud-cli
   ```

3. **检查安装路径**
   ```bash
   which jdc
   ```
   - 确保安装路径在 PATH 环境变量中

### 问题3：JSON 参数解析错误

**现象**：包含 JSON 参数的命令执行失败。

**解决方案**：

1. **正确使用引号**
   ```bash
   # Linux/macOS
   --dimensions '{"key":"value"}'
   
   # Windows PowerShell
   --dimensions '{\"key\":\"value\"}'
   
   # Windows CMD
   --dimensions "{\"key\":\"value\"}"
   ```

2. **验证 JSON 格式**
   - 使用在线工具验证 JSON 格式
   - 确保键名和字符串值使用双引号

## 自定义监控问题

### 问题1：自定义数据上报失败

**现象**：调用 `put-metric-data` 返回错误。

**解决方案**：

1. **检查命名空间**
   - 命名空间只能包含字母、数字、下划线、中划线
   - 不能以数字开头

2. **检查指标名称**
   - 指标名称只能包含字母、数字、下划线、点号
   - 不能以数字或点号开头

3. **检查维度**
   - 维度键名只能包含字母、数字、下划线
   - 维度值长度限制为 256 字符

4. **检查时间戳**
   - 时间戳必须在最近 2 小时内
   - 格式为 ISO8601

### 问题2：自定义数据查询不到

**现象**：上报数据成功，但查询不到。

**解决方案**：

1. **等待数据入库**
   - 自定义监控数据有 1-2 分钟延迟
   - 等待后重试查询

2. **检查查询参数**
   - 确保 `namespace` 和 `metric-name` 与上报时一致
   - 检查时间范围是否包含上报时间

3. **检查维度匹配**
   - 查询时的维度必须与上报时完全匹配
   - 或使用正确的维度过滤

## 性能问题

### 问题1：API 调用缓慢

**现象**：API 响应时间过长。

**解决方案**：

1. **优化查询范围**
   - 缩小 `start-time` 和 `end-time` 范围
   - 减少单次查询的数据量

2. **使用分页**
   - 设置合适的 `page-size`（建议 20-50）
   - 避免一次返回过多数据

3. **使用缓存**
   - 监控数据变化较慢，可适当缓存查询结果
   - 避免重复查询相同数据

4. **并发控制**
   - 控制并发请求数量
   - 避免触发限流

### 问题2：达到限流

**现象**：返回 `RateLimitExceeded` 错误。

**解决方案**：

1. **降低请求频率**
   - 增加请求间隔时间
   - 使用指数退避重试

2. **批量操作**
   - 使用批量接口替代多次单个查询
   - 合并多个请求

3. **申请提额**
   - 联系京东云技术支持申请提高限流阈值
