# AI Agent & RAG Code Review Guide (Core 2026)

## 0. 范围边界（避免职责重叠）

- 本文档聚焦：**Agent/RAG 应用本身**的安全、质量、可靠性与成本
- Skill 的工程发布流程（版本、发布关卡、组织治理）由 `11_agent_skill_review_system.md` 负责
- 非确定性系统评估统计方法由 `14_nondeterministic_eval_guide.md` 负责
- 对抗性/红队审查方法由 `15_adversarial_review_guide.md` 负责

## 1. 必审维度（按市场价值排序）

1. **安全边界**：提示注入、工具越权、数据泄漏
2. **结果可靠性**：RAG忠实度、错误恢复、降级能力
3. **记忆与状态管理**：上下文安全、状态一致性、历史投毒防护
4. **Prompt工程质量**：提示词版本管理、示例质量、敏感性分析
5. **成本与性能**：token预算、缓存命中、延迟SLO
6. **可观测性**：trace、质量指标、审计日志

## 2. 阻断门禁（P0/P1）

- [ ] Prompt Injection 可绕过系统规则
- [ ] Tool Call 无权限模型（无allowlist或人工确认）
- [ ] RAG 检索未做权限隔离（跨租户/跨用户泄漏）
- [ ] 输出缺乏安全过滤，可能产生高风险内容
- [ ] 模型调用无超时/重试/熔断，故障会级联
- [ ] 关键链路无审计，事故不可追踪
- [ ] 上下文窗口无截断策略，可能导致信息泄漏或服务不可用
- [ ] 长期记忆持久化未做权限隔离与隐私保护
- [ ] 对话历史可被投毒（通过注入历史轮次影响后续决策）
- [ ] 系统提示词可被用户输入覆盖或绕过

## 3. RAG 最小质量标准

- [ ] 检索质量：`Recall@k` / `MRR` 有基线
- [ ] 生成质量：`Faithfulness`（答案忠于检索上下文）
- [ ] 数据新鲜度：索引更新策略明确
- [ ] 权限一致性：检索层与业务权限一致

## 4. Agent 工具安全标准

- [ ] 工具定义声明输入schema
- [ ] 参数在执行前二次校验
- [ ] 高风险工具必须人工确认
- [ ] 关键动作具备幂等和回滚能力

## 5. 记忆与状态管理审查

### 5.1 上下文窗口管理

- [ ] 上下文窗口有截断策略（超出时优先保留系统指令 + 最近N轮）
- [ ] 截断策略不会导致关键安全指令丢失
- [ ] 上下文内容在传输和存储中加密
- [ ] 上下文中不包含其他用户的敏感信息

### 5.2 长期记忆管理

- [ ] 长期记忆持久化有访问控制（仅所属用户/租户可读）
- [ ] 记忆数据有保留期限与自动清理策略
- [ ] 记忆写入有校验（防止注入式记忆投毒）
- [ ] 记忆读取结果被视为不可信输入（经过安全过滤后再使用）

### 5.3 状态一致性

- [ ] 状态损坏有检测机制（校验和/版本号）
- [ ] 状态损坏后有恢复路径（回退到最近有效状态）
- [ ] 分布式场景下状态有一致性保证（最终一致或强一致，视业务需求）
- [ ] 状态迁移有兼容性策略（格式变更时向后兼容）

### 5.4 历史投毒防护

- [ ] 对话历史中的注入内容不影响后续轮次的安全判定
- [ ] 历史轮次的安全上下文与当前轮次独立评估
- [ ] 用户可清除/重置对话历史以消除投毒影响
- [ ] 关键决策不依赖不可验证的历史上下文

## 6. Prompt 工程审查

### 6.1 系统提示词质量

- [ ] 系统提示词有版本管理（变更可追踪、可回滚）
- [ ] 系统提示词与代码分离存储（不在业务代码中硬编码）
- [ ] 系统提示词变更经过评审（与代码变更同等对待）
- [ ] 系统提示词有最小化原则（不包含不必要的权限描述或信息）

### 6.2 Few-shot 示例质量

- [ ] Few-shot 示例覆盖核心业务场景（至少覆盖正常/边界/异常各1例）
- [ ] 示例输出格式与实际期望输出一致
- [ ] 示例不包含敏感信息（真实用户数据、内部配置等）
- [ ] 示例数量与多样性合理（过多增加成本，过少降低质量）

### 6.3 提示词敏感性分析

- [ ] 关键提示词的微小改动（措辞/顺序/格式）对输出的影响已评估
- [ ] 高敏感性提示词有锁定机制（变更需额外审批）
- [ ] 提示词 A/B 测试有统计规范（参见 `14_nondeterministic_eval_guide.md` 第 4.2 节）
- [ ] 提示词变更后的回归测试覆盖核心场景

### 6.4 提示词安全

- [ ] 系统提示词不包含可被提取的敏感信息
- [ ] 用户输入与系统指令有明确分隔（无法通过输入覆盖指令）
- [ ] 提示词模板注入风险已评估（动态拼接的变量经过转义）

## 7. 2026 前沿建议（只保留可落地）

- **Policy-as-Code for Agents**：把工具权限和安全规则代码化并进入CI
- **Eval-as-Gate**：把RAG评估（faithfulness/relevance）作为发布门禁
- **Cost Guardrails**：按请求/会话设置token与预算上限
- **Full-Stack Observability**：统一记录`trace_id`, `latency`, `token`, `cost`, `safety_event`
- **Memory Sandboxing**：对话历史与长期记忆在独立沙箱中处理，防止跨会话污染
- **Prompt Regression Testing**：提示词变更自动触发回归测试，量化输出差异

## 8. 最小量化阈值（默认建议）

- `P0` 安全事件：**0容忍**（命中即阻断）
- RAG 忠实度（Faithfulness）：建议 `>= 0.85`
- 检索命中（Recall@k）：建议 `>= 0.80`
- 关键接口超时：建议 `<= 10s`
- 错误率（5xx/失败率）：建议 `< 1%`
- 预算护栏：超预算请求应自动降级或拒绝
- 上下文截断触发率：建议 `< 5%`（过高说明窗口配置不合理）
- 记忆投毒检测率：建议 `>= 90%`（已知攻击模式）
- 提示词变更回归通过率：建议 `>= 95%`

> 以上阈值声明需附样本量与置信区间，参见 `14_nondeterministic_eval_guide.md`。

## 9. 30分钟快审流程

1. **5分钟**：扫阻断门禁（第2节）
2. **5分钟**：核查工具权限与租户隔离（第4节 + 第3节）
3. **5分钟**：核查记忆/状态管理与Prompt工程（第5节 + 第6节）
4. **5分钟**：核查质量阈值与观测数据（第3节 + 第8节）
5. **10分钟**：输出阻断项与最多5条优化建议

## 10. 输出建议格式

- 先列阻断项（P0/P1）
- 再列高杠杆优化（最多5条）
- 每条包含：风险、证据、修复动作、验收标准
