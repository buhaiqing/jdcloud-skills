# 严重级别矩阵

> 定义巡检报告中每条发现的严重级别评分标准。
> 所有评分最终归一化为 0~1，用于 GCL 质量门评估。

## 评分维度

| 维度 | 权重 | 说明 |
|---|---|---|
| Correctness（正确性） | 0.3 | 指标判断是否准确 |
| Safety（安全性） | 0.3 | 是否存在安全风险 |
| Idempotency（幂等性） | 0.1 | 重复巡检是否产生不同结果 |
| Traceability（可追溯） | 0.1 | 报告是否包含完整上下文 |
| Spec Compliance（合规性） | 0.2 | 是否遵循 runbook 定义 |

## 严重级别

| 级别 | 综合评分 | 颜色 | 行动要求 |
|---|---|---|---|
| **Critical** | < 0.3 | 🔴 | 立即处理，24h 内响应 |
| **Warning** | 0.3 ~ 0.5 | 🟡 | 计划排期处理 |
| **Info** | 0.5 ~ 0.8 | 🔵 | 记录观察 |
| **OK** | ≥ 0.8 | ✅ | 无需处理 |

## 级别判定规则

```yaml
critical:
  conditions:
    - "Safety == 0"                          # 安全严重违规
    - "磁盘使用率 > 90%"                       # 资源即将耗尽
    - "服务不可用（健康检查 0%）"               # 服务中断
    - "存在 0.0.0.0/0 开放 DB/管理端口"        # 高危安全配置
  auto_abort: true                           # GCL 中直接终止

warning:
  conditions:
    - "指标超过 Warning 阈值但未达 Critical"
    - "环比昨日同期增长 > 30%"
    - "连续 3 个采样点持续上升"
    - "规格资源水位 > 60%"

info:
  conditions:
    - "指标正常但有上升趋势"
    - "容量预测 30 天内将达 Warning"
    - "成本优化建议（低利用率）"

ok:
  conditions:
    - "所有指标在正常范围"
    - "无上升趋势"
    - "规格资源水位 < 50%"
```
